# Бизнес: Legacy Development — коммерческая недвижимость

Бэклог, отдельный заход. Тип — **realestate**: владеет и управляет коммерческой недвижимостью. Контракт — [[schema-contract]], план — [[_index]].

«Отгрузка» здесь = **сдача проекта / завершённый ремонт / закрытый оплаченный тикет** (полиморфный fulfillment с полями `unit`, `handoverType`, `ticketRef`).

---

## Зачем в холдинге

Замыкает внутригрупповую сеть: Legacy строит/оснащает объекты (закупка у **Spotlights**), запускает их событиями (**Crisis Theater**). Даёт другой профиль экономики, чем сервис и дистрибуция: длинные активы, арендный доход, capex/opex, заполняемость — богатый материал для дашбордов и прогноза.

## Что отличает realestate (показать в данных)

- **Портфель объектов** (`portfolio.yml`/`units.yml`): площадь, тип, заполняемость, арендаторы, ставка.
- **Доход** — рекуррентная аренда (а не разовые проекты) → отдельная модель для Forecast (близко к recurring/retainer).
- **Капитальные проекты и ремонты** — как проекты со сдачей (fulfillment типа handover).
- **Тикеты управления** — facility-тикеты, «закрытый оплаченный тикет» как единица отгрузки.
- Метрики: NOI, заполняемость, дебиторка по аренде, capex-пайплайн.

## Файлы (предварительно)

`company.yml` (type: realestate, fulfillmentLabel: «Сдача / закрытые тикеты»), `portfolio.yml`, `projects.yml` (capex/ремонты), `tickets.yml` (или внутри fulfillments), `clients.yml` (арендаторы), `vendors.yml` (подрядчики), `staff.yml`, `invoices.yml` (аренда + capex), `payments.yml`, `cashflow.yml`, `targets.yml`, `alerts.yml`, `fulfillments.yml`. Запись в группе + intercompany.
