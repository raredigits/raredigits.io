# Подключение продуктов + консолидация

Финальная фаза: перевести Atlas, Forecast и Columbus с хардкод-«Meridian» на общий датасет холдинга, добавить переключатель сущностей и консолидированные виды. Контракт — [[schema-contract]], план — [[_index]].

## Что сейчас (ретайр Meridian)

Atlas (3 экрана), Forecast (3 экрана), Columbus (4 экрана) хардкодят фиктивную **Meridian** (креативная студия, Sofia Ortiz, EUR) прямо в разметке. Числа согласованы вручную и расходятся между продуктами. Контекст: [[atlas-demo-screens]], [[forecast-demo]], [[columbus]].

После унификации Meridian уходит — каждый продукт читает бизнес(ы) холдинга через `_data/demo/*` и 11ty-шаблоны, как уже делает Corsair.

## На какой бизнес смотрит каждый продукт (предложение)

| Продукт | Уровень | Бизнес | Почему |
|---|---|---|---|
| **Corsair HQ** | холдинг | вся группа + дрилл в бизнес | флагман собственника, консолидация — его смысл |
| **Atlas ERP** | операционный | **Spotlights** | каталог + остатки + отгрузки + сверка — лучший ERP-материал |
| **Forecast** | оба | группа и по бизнесу | прогноз консолидированный и по сущности (multi-entity из роадмапа) |
| **Columbus CRM** | бизнес | **Crisis Theater** | отношения событийного бизнеса — длинный тред, реестр обязательств, сетевой P&L (см. [[columbus]]) |

## Консолидация — механика

- **Переключатель сущностей** в UI: «Вся группа / Crisis Theater / Spotlights / …». Читает `group/entities.yml`, рендер обобщённый по `demo[entity.dataPath]`.
- **Роллап**: при «Вся группа» суммируем помесячный `cashflow` всех активных бизнесов по общим колонкам; staff/AR/AP/targets — аналогично. Цветовая разбивка по бизнесам (цвет из `entities.yml`).
- **Мультивалюта**: бизнесы в локальной валюте → пересчёт в валюту отчётности группы по FX-таблице `group/fx.yml` (см. вопрос 1).
- **Элиминация intercompany**: при консолидации вычитаем внутригрупповые сделки из `group/intercompany.yml` (Spotlights→Crisis Theater и т.п.), иначе выручка задваивается. Это и демо-фича, и повод для поста в духе бренда.

## Технические заметки

- **Вёрстка — по [[ui-foundation]]**: Rare Styles `@v0.6.15` + Rare Charts `@v0.9.7` (обе наши), недостающие классы → `assets/css/modules/app.css` (шим под будущими именами библиотеки, тег `LIB:`, лог в `LibraryUpdate.md`). Per-demo модули (`odoo-fintech.css`/`forecast.css`/`watchtower.css`) сворачиваются; классы прототипов переименовываются под семантику библиотек.
- Atlas/Forecast/Columbus сейчас — самодостаточные HTML без 11ty-layout, данные инлайн. Перевод на `_data` = вынести числа в YAML по контракту + прогнать через шаблон (как Corsair). Это устраняет ручную рассинхронизацию — давний пункт из [[atlas-demo-screens]] и [[forecast-demo]] («единый датасет»).
- Forecast-массивы (`const actuals/trend/p50/plan/band` в `<script>`) → в данные сущности; fan-chart остаётся на rare-charts (см. [[forecast-demo]]).
- Columbus: бренд «Meridian» в шапке → бренд холдинга/бизнеса; клиенты — из общего `clients`.
