# UI-фундамент демо-экранов: Rare Styles + Rare Charts

Конвенция вёрстки для **всех** экранов холдинг-демо. Экраны будут переделываться — этот док фиксирует, на чём. План — [[_index]], подключение продуктов — [[wire-up-and-consolidation]].

## Две библиотеки — обе наши

Все экраны проектируем на наших же внешних библиотеках (org `raredigits`), подключаемых с CDN с **зафиксированными** версиями:

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/raredigits/rare-styles@v0.6.15/rare.min.css">
<script src="https://cdn.jsdelivr.net/gh/raredigits/rare-charts@v0.9.7/rare-charts.min.js" defer></script>
```

- **Rare Styles** `@v0.6.15` — дизайн-система (токены, утилиты, компоненты). Канон по использованию — [[markup-rare-styles]] / `_editorial/prompts/markup.md`.
- **Rare Charts** `@v0.9.7` — графики. Канон по API/граблям — [[rare-charts-usage]].

Поскольку библиотеки наши — недостающее не обходим хаком, а **дотачиваем в библиотеку** (через локальный шим + лог, см. ниже).

---

## Недостающие классы → `assets/css/modules/app.css`

Единый локальный модуль для всего, чего в библиотеках пока нет: `assets/css/modules/app.css`. Механизм уже заведён, переиспользуем его (не плодим per-demo модули):

- Примитивы, которых нет в rare-styles, шимятся внизу `app.css` **под их будущими именами в библиотеке** (блок `PROPOSED`), помечаются комментарием `LIB:`.
- Каждый шим логируется в `assets/css/modules/LibraryUpdate.md`: что понадобилось, что сделали локально как заглушку, где она лежит — чтобы вырезать, когда библиотека закроет пробел.
- Порядок подключения: `rare.min.css` → `app.css`.

Цель: всё удачное из `app.css` со временем уезжает в rare-styles / rare-charts, локальный модуль усыхает.

## Сохраняем логику и семантику библиотек

Главное правило при переносе прототипов: **имена и поведение классов следуют семантике библиотек**, а не придумываются заново. Многие классы текущих прототипов под это потребуют **переименования**, прежде чем попасть в новые экраны.

Кандидаты на ревизию (ad-hoc вёрстка вне библиотеки, появившаяся в старых демо):
- Per-demo модули `assets/css/modules/odoo-fintech.css`, `forecast.css`, `watchtower.css` — заменяются единым `app.css` + классами библиотеки.
- Префиксные/локальные классы из прототипов (`.erp-shell`, `.fin-*`, `.kpi-*`, `.corsair-*`, `.tct-*` и т.п.) — мапятся на семантику rare-styles либо легализуются как `LIB:`-шимы под библиотечными именами.
