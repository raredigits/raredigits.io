# Демо: Odoo, переосмысленный в Rare Styles

Интерактивные HTML-мокапы нескольких экранов Odoo, переделанных так, будто их собрал современный финтех/необанк. Иллюстрация нашего подхода: берём привычный ERP → накрываем его нашей дизайн-библиотекой и представлением о том, как бизнес-информация должна подаваться лицам, принимающим решения → на выходе продукт, у которого под капотом знакомое решение, а сверху — то, на что собственник готов смотреть.

Это ровно тот тезис, который компания продаёт: *«мы садимся поверх существующего ERP/CRM/учётки клиента и делаем цифры читаемыми для тех, кто владеет бизнесом»*. Демо делает этот тезис наглядным на самом узнаваемом ERP рынка. Дополнительный бонус — попутно витрина возможностей **Rare Styles**.

> Контекст для своих: Atlas ERP (наш бывший флагман) — это форк Odoo. Так что «улучшить Odoo нашими руками» — не абстракция, а наша же история.

---

## Цель и формат

- **Что:** 3 экрана (расширяемо до 5–6), каждый — отдельный самодостаточный интерактивный HTML-мокап.
- **Зачем:** презентационная иллюстрация подхода (сайт / питч / кейс). Не продакшн-приложение.
- **Состояние:** мокапы должны быть «живыми» — их можно открыть в браузере, подвигать, поправить стили и довести до презентационного вида.
- **Стиль:** абстрактный модный необанк. Не копируем конкретный банк, не брендируем под реального клиента. Данные — синтетические, но правдоподобные.

### Не делаем

- Не используем классы и наработки из `demo/corsair/` — то демо требует отдельного ревью и переписи, оно **не** на нашей библиотеке. Здесь чистый старт на Rare Styles.
- Не воспроизводим уродство Odoo «как есть». Цель — показать идеал, а не «до/после» с акцентом на «до» (хотя пару слов «что было» в подписи к кейсу дать можно).
- Никаких buzzwords из стайлгайда в текстах интерфейса.

## Выбор экранов

Критерии: **(1)** узнаваемость как Odoo, **(2)** разнообразие view-паттернов (чтобы показать систему, а не один трюк), **(3)** попадание в нарратив «деньги → решения».

| # | Экран | Odoo-оригинал | View-паттерн | Что демонстрирует |
|---|---|---|---|---|
| 1 | **Дашборд собственника** | Accounting / Spreadsheet Dashboard | Dashboard | Большие числа как герои, KPI, статусы, тренды. Флагман демо. |
| 2 | **Счёт (Invoice)** | `account.move` form view | Form / документ | Самый «денежный» и узнаваемый документ. Степпер статуса, чистый платёжный лист. |
| 3 | **Воронка продаж** | CRM Pipeline | Kanban | Колонки-стадии, карточки сделок, плотность без шума. |

Опционально позже: банковская сверка (reconcile — самый финтех-flow), список счетов (list view), карточка контакта (form + активности).

**Для первой итерации делаем экран 1 (дашборд)** как референс-эталон — на нём обкатываем модуль стилей и направление, остальные тиражируем по утверждённому паттерну.

## Принципы Rare Styles (на чём строим)

Библиотека уже подключена на сайте:

```
https://cdn.jsdelivr.net/gh/raredigits/rare-styles@v0.6.15/rare.min.css
```

### Токены, которые задают «лицо»

- **Акцент:** фирменный зелёный `--brand-color: #00ff4e` (+ `--brand-color-dark #00b300`, `--brand-color-light #d8f9d1`). Используем **скупо** — один акцент на экран, на главном действии/положительном статусе.
- **Семантика:** `--blue #1677ff`, `--green #389e0d`, `--red #f00`, `--orange #fa8c16`, `--yellow #ffe000`. Цвет — только для статуса, не для украшения.
- **Нейтральные:** `--gray-lightest #fafafa`, `--gray-light #f0f0f0`, `--gray-mid-light #e4e4e4`, `--gray #ccc`, `--gray-trans #888`, `--gray-dark #333`, `--gray-darkest #191919`. Канва — почти-белая или тёмная (`gray-darkest`), границы — волосяные (1px `--border-color`).
- **Типографика:** заголовки `--heading-font: 'Playfair Display'` (serif), текст `--primary-font: 'Fira Sans'`, **числа — `--code-font: 'Cousine'` mono** (табличное выравнивание разрядов — критично для финансов). Лёгкие начертания: `--font-weight-light: 200`.
- **Ритм:** база 16px. Шкала `--space-xs 4 / sm 8 / md 16 / lg 32 / xl 48 / xxl 96`. Много воздуха.
- **Форма:** радиус скромный — `--space-xs` (4px) у карточек; кнопки — pill (`--space-lg`). Тени мягкие (`--shadow-sm/md/lg`).

### Полезные классы, которые уже есть

`card`, `card-dashboard-bordered`, `card-hover`, `card-inverted`, `card-grid` (+`--cols-2/3/4`), `grid-cols-N`, `col-span-N`, `table-numeric/striped/bordered/small`, `tag`, `callout`, `button`, `lead`, цветовые `*-bg`/`*-link`, шкалы `air-*`/`gap-*`/`margin-*`.

### Философия подачи (наша)

1. **Решение, а не данные.** Сверху — то, что нужно решить сегодня; детали — ниже / по клику. Иерархия диктуется вопросом собственника, а не структурой БД Odoo.
2. **Число — герой.** Главная метрика крупно, моноширинным, с понятной подписью человеческим языком («сколько можно реально потратить», а не «Unrestricted Cash Balance»).
3. **Цвет = смысл.** Зелёный/красный только там, где это статус. Остальное — нейтраль.
4. **Воздух вместо линий.** Группируем пространством и типографикой, а не рамками и фоновыми заливками (анти-Odoo).
5. **Тон интерфейса — наш.** Дружелюбно-ироничный, по-человечески. Reader — peer, не студент.

## Чего в библиотеке нет → собираем в модуль

Для дашбордов в Rare Styles пока нет готовых компонентов. Собираем их отдельным модулем:

```
assets/css/modules/odoo-fintech.css
```

Модуль строится **только на токенах библиотеки** (`var(--…)`), без хардкода цветов/отступов — чтобы при обновлении rare-styles всё поехало вместе, и чтобы удачные компоненты можно было поднять обратно в ядро библиотеки.

Нужные компоненты:

- **KPI-плитка** — крупное число + дельта/тренд + подпись.
- **Бейдж-статус (pill)** — `paid / overdue / draft / on-track / at-risk` в семантических цветах.
- **Прогресс-бар** — план/факт.
- **Спарклайн / мини-бар-чарт** — тренд без осей (инлайн-SVG или CSS-бары).
- **Лента транзакций** — строка: иконка категории + контрагент + сумма (моно, выровнено по правому краю) + статус.
- **Канбан** — колонки-стадии + карточки сделок (drag не обязателен; визуал — да).
- **Сегмент-контрол / тоггл** — переключатель периодов/видов.
- **Аватар** — инициалы в круге.
- **Chrome приложения** — топ-бар + сайдбар-лаунчер (узнаваемая «оболочка ERP», но в нашей подаче).

Каждый компонент именуем по БЭМ-конвенции библиотеки (`block`, `block__el`, `block--mod`), префикс модуля по необходимости.

## Технические решения

- **Расположение мокапов:** `demo/atlas/<screen>/index.html` (отдаются дев-сервером 11ty по `/demo/atlas/<screen>/`).
- **Самодостаточность:** каждый файл — полный HTML-документ со своим `<head>`, подключает CDN rare-styles + `/assets/css/modules/odoo-fintech.css`. Без 11ty-layout, чтобы открывался и правился изолированно.
- **Иконки:** Material Symbols через Google Fonts CDN (как уже принято в demo) либо инлайн-SVG. Решить на первом экране.
- **Темы:** делаем light по умолчанию; тёмный вариант (`gray-darkest` канва) — как опция-витрина, если успеваем.
- **Данные:** синтетические, захардкожены в разметку (это мокап, не шаблон).
